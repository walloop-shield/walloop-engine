# Security Policy

## Supported Versions

Security fixes are applied to the `main` branch.

## Reporting a Vulnerability

If you find a security issue, do not open a public issue.

Send details to: `contact@walloop.io`

Include:
- Affected component and version/commit
- Reproduction steps
- Impact assessment
- Suggested mitigation (if available)

## Response SLA

- Initial acknowledgement: up to 3 business days
- Triage and severity classification: up to 7 business days
- Fix timeline: depends on severity and operational risk

## Disclosure Policy

- Coordinate disclosure with the Walloop team
- Public disclosure should happen only after patch availability
- Credentials/tokens exposed historically must be rotated immediately
